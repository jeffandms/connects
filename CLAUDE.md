# Professional impact repo guidance

This repository is Jeff Anderson's working system for tracking professional impact, Connect inputs, promotion evidence, and manager-ready summaries.

## Core goal

Keep this repo current with concrete evidence of work completed, progress made, leverage created, and growth shown over time.

## Primary artifacts to maintain

Update these files whenever new meaningful information is learned:

1. `docs\tracker\current-focus.md` - active work, risks, blockers, and near-term priorities.
2. `docs\tracker\impact-log.md` - dated evidence entries with enough detail to reuse later.
3. `docs\tracker\promotion-case.md` - synthesized case organized around the themes Jeff's boss values.
4. `docs\connects\next-connect.md` - working draft for the next Connect, only when preparing or when updates materially change the narrative.

## Boss-valued themes

Classify evidence under one or more of these exact themes:

- `net-new-dataverse`
- `legacy-reliability`
- `cogs`
- `team-growth`

## Source priority

Use sources in this order:

1. Direct user-provided notes, pasted context, and requests in the current session.
2. Existing repo files in `docs\tracker\` and `docs\connects\`.
3. Historical Connect PDFs and guidance PDFs in the repo.
4. WorkIQ sources such as email, Teams, meetings, and files only when the user explicitly asks for latest evidence or asks to use WorkIQ, and only after any required consent or EULA step is satisfied.

## Evidence standards

- Prefer shipped work, released features, adoption, unblockings, escalations resolved, customer impact, design influence, mentoring, and cross-team leverage.
- Keep claims concrete. Include names of systems, customers, capabilities, metrics, or timelines when known.
- Summarize sensitive mail or chat content instead of copying long excerpts.
- When an item is interesting but incomplete, log it with the open question or follow-up needed.

## Update behavior

When updating the repo:

1. Add or refresh active items in `current-focus.md`.
2. Append new dated entries to `impact-log.md`.
3. Strengthen the appropriate sections in `promotion-case.md`.
4. Refresh `next-connect.md` when the user asks for Connect prep or when the update obviously changes the review narrative.

## Default operating rhythm

- Weekly or biweekly: run the weekly update workflow.
- After a notable win, escalation, customer impact, design review, or mentoring moment: capture it immediately.
- Before 1:1s, skip-levels, or review checkpoints: refresh the manager brief or Connect draft.
