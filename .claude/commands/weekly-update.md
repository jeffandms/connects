Refresh this repo with the latest professional-impact evidence.

Use `CLAUDE.md` as the operating policy. Treat `$ARGUMENTS` as the requested time window or any extra guidance.

## Goals

- Keep `docs\tracker\current-focus.md` current.
- Append new concrete evidence to `docs\tracker\impact-log.md`.
- Strengthen `docs\tracker\promotion-case.md`.
- Update `docs\connects\next-connect.md` only if the new information changes the likely review narrative.

## Workflow

1. Read the current tracker and Connect files before changing anything.
2. Use notes from the user first.
3. If the user asks for "latest" evidence or explicitly asks to use WorkIQ, gather the most relevant email, Teams, meetings, and file context from the requested window.
4. Extract only meaningful items: shipped work, unblockings, customer impact, reliability wins, COGS wins, mentoring, cross-team influence, or decision-shaping work.
5. Classify each item under one or more of:
   - `net-new-dataverse`
   - `legacy-reliability`
   - `cogs`
   - `team-growth`
6. Update the tracker docs in that order: current focus, impact log, promotion case, then Connect draft if needed.
7. End with a short summary of what changed and the most important evidence gaps to capture next time.
